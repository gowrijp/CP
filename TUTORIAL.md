<h3><a id="user-content-vector" class="anchor" aria-hidden="true" href="#vector"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Vector</h3>
<p>A vector is similar to an array but with dynamic size where elements can be added from one end.</p>
<p><strong>Design:</strong> The values are stored consecutively in memory like array for O(1) element access. Costly O(n) for insertion/deletion in the middle of vector.</p>
<div class="highlight highlight-source-c++"><pre><span class="pl-c"><span class="pl-c">//</span> declaration and initialization</span>
vector&lt;<span class="pl-k">int</span>&gt; vec; <span class="pl-c"><span class="pl-c">//</span> vec is a vector variable</span>
vector&lt;<span class="pl-k">int</span>&gt; <span class="pl-en">vec_1</span> (<span class="pl-c1">5</span>); <span class="pl-c"><span class="pl-c">//</span> vec_1 is a vector of 0, 0, 0, 0, 0</span>
vector&lt;<span class="pl-k">int</span>&gt; <span class="pl-en">vec_2</span> (<span class="pl-c1">4</span>,<span class="pl-c1">100</span>); <span class="pl-c"><span class="pl-c">//</span> vec_2 is a vector of 100, 100, 100, 100</span>

vector&lt;<span class="pl-k">int</span>&gt;::iterator vec_iter; <span class="pl-c"><span class="pl-c">//</span> vec_iter is an iterator for vector&lt;int&gt;</span>
vec_iter = vec_2.begin(); <span class="pl-c"><span class="pl-c">//</span> vec_2.begin() is the address of its first element, *vec_iter contains 100</span>

vector&lt;<span class="pl-k">int</span>&gt; <span class="pl-en">vec_3</span> (vec_2.begin(), vec_2.end()); <span class="pl-c"><span class="pl-c">//</span> initializes from another vector</span>
vector&lt;<span class="pl-k">int</span>&gt; <span class="pl-en">vec_4</span> (vec_3); <span class="pl-c"><span class="pl-c">//</span> copies entire vec_3 vector</span>

<span class="pl-k">int</span> arr[] = {<span class="pl-c1">1</span>,<span class="pl-c1">2</span>,<span class="pl-c1">3</span>};
vector&lt;<span class="pl-k">int</span>&gt; <span class="pl-en">vec_5</span> (arr, arr+<span class="pl-c1">2</span>); <span class="pl-c"><span class="pl-c">//</span> initializes from arr array, vec_5 is a vector of 1, 2</span>

<span class="pl-c"><span class="pl-c">//</span> size() - O(1) - returns size of vector</span>
<span class="pl-k">int</span> vec_size = vec_5.size(); <span class="pl-c"><span class="pl-c">//</span> vec_size contains 2</span>

<span class="pl-c"><span class="pl-c">//</span> empty() - O(1) - returns 1 if vector is empty, 0 otherwise</span>
<span class="pl-k">bool</span> vec_empty = vec.empty(); <span class="pl-c"><span class="pl-c">//</span> vec_empty contains 1</span>

<span class="pl-c"><span class="pl-c">//</span> access - O(1)</span>
<span class="pl-k">int</span> value = vec_5[<span class="pl-c1">0</span>]; <span class="pl-c"><span class="pl-c">//</span> similar to array, value contains 1</span>

<span class="pl-c"><span class="pl-c">//</span> front() - O(1) - returns reference to first element</span>
value = vec_5.front(); <span class="pl-c"><span class="pl-c">//</span> value contains 1 </span>

<span class="pl-c"><span class="pl-c">//</span> back() - O(1) - returns reference to last element</span>
value = vec_5.back(); <span class="pl-c"><span class="pl-c">//</span> value contains 2 </span>

<span class="pl-c"><span class="pl-c">//</span> push_back() - O(1) - inserts new element at the end</span>
vec_5.push_back(<span class="pl-c1">3</span>); <span class="pl-c"><span class="pl-c">//</span> vec_5 contains 1, 2, 3</span>

<span class="pl-c"><span class="pl-c">//</span> pop_back() - O(1) - removes last element</span>
vec_5.pop_back(); <span class="pl-c"><span class="pl-c">//</span> vec_5 contains 1, 2</span>

<span class="pl-c"><span class="pl-c">//</span> insert() -  inserts elements at desired positions. Moves the elements after "position" accordingly.</span>
<span class="pl-c"><span class="pl-c">//</span> insert(position, value) - O(n-position) - due to moving elements after position</span>
vec_5.insert(vec_5.begin()+<span class="pl-c1">1</span>, <span class="pl-c1">4</span>); <span class="pl-c"><span class="pl-c">//</span> vec_5 contains 1, 4, 2</span>

<span class="pl-c"><span class="pl-c">//</span> insert(position, repetitions, value) - O(repetitions+n-position)</span>
vec_5.insert(vec_5.begin(), <span class="pl-c1">3</span>, <span class="pl-c1">2</span>); <span class="pl-c"><span class="pl-c">//</span> vec_5 contains 2, 2, 2, 1, 4, 2</span>

<span class="pl-c"><span class="pl-c">//</span> insert(position, start, end) - O(end-start+n-position)</span>
vec_5.insert(vec_5.begin(), vec_2.begin(), vec_2.begin()+<span class="pl-c1">2</span>); <span class="pl-c"><span class="pl-c">//</span> vec_5 contains 100, 100, 2, 2, 2, 1, 4, 2</span>

<span class="pl-c"><span class="pl-c">//</span> assign() - overwrites values from the beginning</span>
<span class="pl-c"><span class="pl-c">//</span> assign(repetitions, value) - O(n') where n' is final size</span>
vec_5.assign(<span class="pl-c1">1</span>, <span class="pl-c1">5</span>); <span class="pl-c"><span class="pl-c">//</span> vec_5 contains 5</span>

<span class="pl-c"><span class="pl-c">//</span> assign(start, end) - O(n') - n' is final size</span>
vec_5.assign(vec_2.begin(), vec_2.begin()+<span class="pl-c1">3</span>); <span class="pl-c"><span class="pl-c">//</span> vec_5 contains 100, 100, 100</span>

<span class="pl-c"><span class="pl-c">//</span> erase() - erases elements at desired positions</span>
<span class="pl-c"><span class="pl-c">//</span> erase(position) - O(n-position) - due to moving elements after position</span>
vec_5.erase(vec_5.begin()); <span class="pl-c"><span class="pl-c">//</span> vec_5 contains 100, 100</span>

<span class="pl-c"><span class="pl-c">//</span> erase(start, end) - O(end-start+n-end) - due to elements erased and moving </span>
vec_5.erase(vec_5.begin(), vec_5.begin()+<span class="pl-c1">1</span>); <span class="pl-c"><span class="pl-c">//</span> vec_5 is 100</span>

<span class="pl-c"><span class="pl-c">//</span> swap() - O(1) - swaps the contents of vectors</span>
vec_1.swap(vec_2); <span class="pl-c"><span class="pl-c">//</span> vec_1 contains 100, 100, 100, 100 and vec_2 contains 0, 0, 0, 0, 0</span>

<span class="pl-c"><span class="pl-c">//</span> clear() - O(n) delete the vector</span>
vec_5.clear(); <span class="pl-c"><span class="pl-c">//</span> vec_5 is empty</span>

<span class="pl-c"><span class="pl-c">//</span> relational - O(n)</span>
<span class="pl-k">bool</span> status;
status = (vec_3 == vec_4); <span class="pl-c"><span class="pl-c">//</span> status contains 1</span>
status = (vec_3 != vec_4); <span class="pl-c"><span class="pl-c">//</span> status contains 0</span>
status = (vec_1 &lt; vec_2); <span class="pl-c"><span class="pl-c">//</span> status contains 0, lexicographically compares elements from left</span></pre></div>
<h3><a id="user-content-deque" class="anchor" aria-hidden="true" href="#deque"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Deque</h3>
<p>A deque is double ended queue similar to vector with an additional benefit where elements can be added from both start and end.</p>
<p><strong>Design:</strong> Contiguous memory is not guaranteed unlike vector and insertion/deletion in the middle is costly. However the access is still O(1).</p>
<div class="highlight highlight-source-c++"><pre><span class="pl-c"><span class="pl-c">//</span> declaration and initialization</span>
deque&lt;<span class="pl-k">int</span>&gt; deq; <span class="pl-c"><span class="pl-c">//</span> deq is a deque variable</span>
deque&lt;<span class="pl-k">int</span>&gt; <span class="pl-en">deq_1</span> (<span class="pl-c1">5</span>); <span class="pl-c"><span class="pl-c">//</span> deq_1 is a deque of 0, 0, 0, 0, 0</span>
deque&lt;<span class="pl-k">int</span>&gt; <span class="pl-en">deq_2</span> (<span class="pl-c1">4</span>,<span class="pl-c1">100</span>); <span class="pl-c"><span class="pl-c">//</span> deq_2 is a deque of 100, 100, 100, 100</span>

deque&lt;<span class="pl-k">int</span>&gt;::iterator deq_iter; <span class="pl-c"><span class="pl-c">//</span> deq_iter is an iterator for deque&lt;int&gt;</span>
deq_iter = deq_2.begin(); <span class="pl-c"><span class="pl-c">//</span> deq_2.begin() is the address of its first element, *deq_iter contains 100</span>

deque&lt;<span class="pl-k">int</span>&gt; <span class="pl-en">deq_3</span> (deq_2.begin(), deq_2.end()); <span class="pl-c"><span class="pl-c">//</span> initializes from another deque</span>
deque&lt;<span class="pl-k">int</span>&gt; <span class="pl-en">deq_4</span> (deq_3); <span class="pl-c"><span class="pl-c">//</span> copies entire deq_3 deque</span>

<span class="pl-k">int</span> arr[] = {<span class="pl-c1">1</span>,<span class="pl-c1">2</span>,<span class="pl-c1">3</span>};
deque&lt;<span class="pl-k">int</span>&gt; <span class="pl-en">deq_5</span> (arr, arr+<span class="pl-c1">2</span>); <span class="pl-c"><span class="pl-c">//</span> initializes from arr array, deq_5 is a deque of 1, 2</span>

<span class="pl-c"><span class="pl-c">//</span> size() - O(1) - returns size of deque</span>
<span class="pl-k">int</span> deq_size = deq_5.size(); <span class="pl-c"><span class="pl-c">//</span> deq_size contains 2</span>

<span class="pl-c"><span class="pl-c">//</span> empty() - O(1) - returns 1 if deque is empty, 0 otherwise</span>
<span class="pl-k">bool</span> deq_empty = deq.empty(); <span class="pl-c"><span class="pl-c">//</span> deq_empty contains 1</span>

<span class="pl-c"><span class="pl-c">//</span> access - O(1)</span>
<span class="pl-k">int</span> value = deq_5[<span class="pl-c1">0</span>]; <span class="pl-c"><span class="pl-c">//</span> similar to array, value contains 1</span>

<span class="pl-c"><span class="pl-c">//</span> front() - O(1) - returns reference to first element</span>
value = deq_5.front(); <span class="pl-c"><span class="pl-c">//</span> value contains 1 </span>

<span class="pl-c"><span class="pl-c">//</span> back() - O(1) - returns reference to last element</span>
value = deq_5.back(); <span class="pl-c"><span class="pl-c">//</span> value contains 2 </span>

<span class="pl-c"><span class="pl-c">//</span> insert() - inserts elements at desired positions</span>
<span class="pl-c"><span class="pl-c">//</span> insert(position, value) - O(n-position) due to moving elements after position</span>
deq_5.insert(deq_5.begin()+<span class="pl-c1">1</span>, <span class="pl-c1">4</span>); <span class="pl-c"><span class="pl-c">//</span> deq_5 contains 1, 4, 2</span>

<span class="pl-c"><span class="pl-c">//</span> insert(position, repetitions, value) - O(repetitions+n-position)</span>

<span class="pl-c"><span class="pl-c">//</span> insert(position, start, end) - O(end-start+n-position)</span>

<span class="pl-c"><span class="pl-c">//</span> push_front() - O(1) - inserts new element at the start</span>

<span class="pl-c"><span class="pl-c">//</span> pop_front() - O(1) - removes first element</span>

<span class="pl-c"><span class="pl-c">//</span> erase() - erases elements at desired positions</span>
<span class="pl-c"><span class="pl-c">//</span> erase(position) - O(n-position)</span>
deq_5.erase(deq_5.begin()); <span class="pl-c"><span class="pl-c">//</span> deq_5 contains 100, 100</span>

<p>A doubly linked list with discontiguous storage.</p>
<p><strong>Design:</strong> Allows constant insertion/deletion anywhere. O(position) time for accessing.</p>

<span class="pl-k">int</span> mulmap_size = mulmap_1.size(); <span class="pl-c"><span class="pl-c">//</span> mulmap_size contains 3</span>

<span class="pl-c"><span class="pl-c">//</span> insert() - inserts new elements into the multimap</span>
<span class="pl-c"><span class="pl-c">//</span> insert(pair) - O(log n)</span>

mulmap.erase(mulmap_iter); <span class="pl-c"><span class="pl-c">//</span> mulmap is empty</span>

<span class="pl-c"><span class="pl-c">//</span> find(key) - O(log n) - returns iterator to element if found, end() otherwise</span>

mulmap_iter = mulmap_1.find(<span class="pl-c1">1</span>); <span class="pl-c"><span class="pl-c">//</span> mulmap_iter is mulmap_1.end()</span>

<span class="pl-c"><span class="pl-c">//</span> count(key) - O(log n + num) returns count of key</span>

<div class="highlight highlight-source-c++"><pre><span class="pl-c"><span class="pl-c">//</span> declaration and initialization</span>
